Function overview

Everything openWCS does, honestly tagged.

A single map of the warehouse functions openWCS covers. Each card says plainly whether it is Built today or on the Roadmap — so you can tell what you can run now from what is designed but not yet implemented.

Built today

The functions that already run.

These are implemented in the engine: storage and movement logic, allocation and planning, and the conveyor control plane. Follow a card for the deep dive.

Slotting
Built

Put-away engine

A weighted, configurable put-away scorer places each SKU by velocity, lane, redundancy and aisle balance — with self-taught ABC velocity.
Replenishment
Built

Min/max & off-peak top-off

Min/max refills, opportunistic off-peak top-off to max, and direct-to-pick cross-docking when a forward face has headroom.
Conveyors
Built

Routing & self-learning

Vendor-neutral topology graph, per-scan next-hop pathfinding, loop limits, PLC controllers, and topology discovery from sniffed scan traffic.
ASRS
Built

Automated storage logic

Block-level put-away, re-slotting, in-aisle depth and dual-cycle behaviour, plus empty-HU management for shuttle / crane / AutoStore / AMR-GTP.
Allocation
Built

Allocation & pick planning

Stock allocation, multi-size cubing and batch-pick planning are built. Operator pick confirmation and GTP workflows are roadmap — see the page.
Cycle counting
Roadmap

Stock counting

A COUNT order type, a StockAdjusted projection and a sample BPMN skeleton exist; the counting tasks, variance counts and reconciliation UI are roadmap.

On the roadmap

Designed, not yet shipped.

These have foundations in the codebase but are not implemented end-to-end. They are framed as planned so you always know where the line is.

Picking
Roadmap

Pick execution

Today only the planning exists. Operator / goods-to-person pick confirmation, pick-by-light/voice/RF and GTP-station workflow are planned.
Cycle counting
Roadmap

Counts & reconciliation

Counting tasks, blind and variance counts, reconciliation and the count UI are planned on top of the existing COUNT order and StockAdjusted projection.
KPI dashboards
Roadmap

Operational metrics

Throughput, dwell, utilisation and SLA dashboards are planned. There is no metrics store or dashboard layer yet.
Hardware viz
Roadmap

Live equipment status

Today only the conveyor topology editor exists. Live equipment / ASRS / AMR / AutoStore status visualisation is planned.
