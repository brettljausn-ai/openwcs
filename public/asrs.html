Deep dive · ASRS & goods-to-person

Slotting that balances the floor.

In a shuttle or crane ASRS, AutoStore, or an AMR goods-to-person system, you don't pick from a fixed shelf — the system brings handling units to a station. So openWCS slots a SKU to the block as a whole (the pool across all aisles) and decides the actual location per unit at put-away, balancing four objectives that genuinely pull against each other.

The put-away decision

One weighted score, four objectives.

Each candidate location is scored; hard constraints filter first (lane capacity, one SKU per deep lane, and a cap on how much of a SKU may sit in one aisle). The weights are per-block, so you tune the trade-off rather than living with a vendor's hidden rules.

Velocity

Fast movers near the exit

Each location has a distance-to-exit. Class-A SKUs are pulled toward the aisle port for short cycles; slow movers settle deep. ABC class is taught in per SKU.

Multi-deep

Same SKU per lane

A triple-deep lane (X1,Y1,Z1–3) holds one SKU so units never block each other. Partial same-SKU lanes are filled first to avoid honeycombing — empty-but-unusable depth.

Redundancy

Survive an aisle outage

A SKU is deliberately spread across aisles (a min-aisle floor + a max-share-per-aisle cap) so a single shuttle/crane going offline can't strand it — and it can be picked in parallel.

Balance

Even out the aisles

Put-away prefers emptier aisles so fill levels — and therefore travel and throughput — stay balanced, instead of one aisle becoming the hot-spot bottleneck.

Direct-to-pick

Cross-dock when it fits

If a forward pick face for the SKU has headroom below its max, inbound is routed straight there — skipping reserve and a later replenishment trip.

Re-slotting

Correct drift off-peak

During idle windows the same scorer re-evaluates stored units and recommends moves as velocity drifts, lanes fragment, or aisles skew — keeping the layout optimal over time.

Two worlds, one engine

Automated blocks and manual pick faces.

Automated blocks slot to the pool (random/dynamic storage); manual racking and pick-by-light faces slot a SKU+UoM to a fixed location with min/max, so pickers rely on stable, learnable positions.

Pick faces

Fixed slots, min/max

Assign SKU+UoM to a location with a minimum and maximum. Below min raises a refill — emergency if the face is empty, scheduled otherwise.

Replenishment

Opportunistic top-off

In off-peak windows every face is topped up to max, so disruptive hot replenishments during peak are avoided.

Goods-to-person

HUs to the station

For GTP blocks, the WCS retrieves handling units to a pick station and returns them — the operator never travels into storage.

At a glance

Receipt → scored location.

  receipt (SKU, HU) ─►  openWCS slotting
                         │  direct-to-pick?  → forward face (cross-dock)
                         │  else score block pool:
                         │     velocity-to-exit · same-SKU lane · redundancy · balance
                         │     ── hard: lane full · one SKU/lane · max %/aisle ──
                         ▼
                    chosen location ─►  put-away move  (→ GTP pick station on demand)
  off-peak: replenish faces to max · re-slot stored HUs toward better locations

Configurable & open

Slot it your way.
