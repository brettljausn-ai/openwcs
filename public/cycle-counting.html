Function · Cycle counting Roadmap

Counting, scaffolded.

Cycle and stock counting keep recorded inventory honest. The foundations exist in openWCS — but the working count cycle is designed, not yet implemented.

⚑ Roadmap — planned, not yet implemented. A COUNT order type, a StockAdjusted projection and a sample BPMN skeleton are in place as the foundation.

What exists today

The foundations are in the codebase.

These pieces are present already — the order, the event projection and a process skeleton — so the roadmap builds on real groundwork rather than a blank page.

COUNT order
Built

An order type to count

A dedicated COUNT order type exists, giving counting work a first-class place in the order model rather than bolting it on later.

Projection
Built

StockAdjusted events

A StockAdjusted projection is in place, so an adjustment arising from a count has a defined event and read model to land in.

BPMN skeleton
Built

A sample process

A sample BPMN skeleton sketches the count flow, ready to grow into the real orchestration that drives tasks and reconciliation.

On the roadmap

The working count cycle.

Turning the scaffold into an operable function: generating tasks, capturing counts without bias, resolving variances and giving operators a UI.

Tasks
Roadmap

Counting tasks

Planned: generating and dispatching counting tasks — by location, by velocity class or on a cycle — from the COUNT order. Not yet implemented.

Count modes
Roadmap

Blind & variance counts

Planned: blind counts that hide the expected quantity, plus variance counts that re-count only where the first pass disagreed. Designed for, not yet built.

Reconciliation
Roadmap

Resolve & adjust

Planned: reconciliation that compares counted vs recorded, posts a StockAdjusted event within tolerance and escalates the rest. Not yet implemented.

UI
Roadmap

Operator & review screens

Planned: the count UI for operators to enter counts and for supervisors to review variances and approve adjustments. Designed, not yet built.
